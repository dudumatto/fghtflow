# Formato de saída (consistente e escaneável)

Use quando a tarefa for complexa (múltiplos arquivos, risco, várias áreas).

```md
# Resultado

## Status
✅ Concluído | ⚠️ Parcial | ❌ Bloqueado

## Resumo
- (1–5 bullets)

## Agentes usados
- Design/UI
- Frontend
- Backend
- QA/Test
- Security
- Documentation

## Problemas encontrados
- (prioridade alta → baixa)

## Correções
- arquivo: o que mudou (curto)

## Validação
- comandos / checklist

## Riscos
- o que ainda pode dar errado
```
