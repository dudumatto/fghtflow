# Skill: Documentação

## Objetivo
Docs curtas que realmente ajudam alguém a usar/manter o projeto.

## Práticas (curtas)
- Documentar “como rodar” e “como validar”.
- Exemplos mínimos (1 request, 1 payload, 1 env var).
- Registrar decisões não óbvias (trade-offs).
- Manter docs próximas do código (evitar wiki solta).
