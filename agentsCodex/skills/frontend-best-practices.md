# Skill: Frontend Best Practices

## Objetivo
Código de UI previsível, acessível e fácil de manter.

## Práticas (curtas)
- Estado sempre com `loading/erro/sucesso`.
- Evitar duplicação: extrair componentes/handlers repetidos.
- Validar inputs no cliente, mas **não confiar** nele.
- Tratamento de erros: mensagens úteis + fallback de UI.
- Acessibilidade mínima: foco visível, labels, teclado.
- Performance: memoização com parcimônia; evitar re-renderizações óbvias.
