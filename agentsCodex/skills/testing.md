# Skill: Testes

## Objetivo
Evitar regressões com o mínimo de testes úteis.

## Práticas (curtas)
- Para bug: 1 teste que falha antes + passa depois (quando possível).
- Cobrir: happy path, erro, bordas.
- Preferir testes de integração para contratos e fluxos críticos.
- Checklist manual curto quando automação não existir.
